# Congratulations!

You've successfully added your smart contract, set up the environment, and written your first test. You're now ready to build more powerful features on top of it!

## What's Next?

Your contract is working locally, but there's a journey ahead before it runs on mainnet.

### Deepen Your Knowledge

Explore the advanced topics in [Contract Development](../smart-contract/overview.md):

- [Cross-Contract Calls](../smart-contract/cross-inner-call.md) — call other contracts from yours
- [Assets and Shares](../smart-contract/assets-and-shares.md) — manage asset ownership and transfers
- [States](../smart-contract/states.md) — persistent data management

### Validate Your Contract

- Run the [SC Verification Tool](../resources/contract-verify-tool.md) to check your code complies with all restrictions
- Set up a [local testnet](../testing/testnet.md) with [Qubic Core Lite](../resources/qubic-lite-core.md) for realistic network testing

### Deploy to Mainnet

Read the [Smart Contract Lifecycle](../lifecycle.md) guide to understand the complete path from code to mainnet — including PR submission, core dev review, computor proposals, voting, and the IPO process.

:::info
A smart contract cannot go live without passing core dev review, winning a computor vote (451+ of 676 computors participating, majority approval), and completing a successful IPO (all 676 shares sold). The minimum timeline from start to go-live is roughly 3 weeks. See the [full lifecycle](../lifecycle.md) for details.
:::
